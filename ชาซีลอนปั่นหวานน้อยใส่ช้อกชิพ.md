# Project Assignment #1

## Target User
- นักดาราศาสตร์ (Astronomer)
- ผู้สังเกตการณ์ทางวิทยาศาสตร์ (Science Observer)
- บุคลากรดำเนินการ (Operations staff)
- ผู้เยี่ยมชม (Visitor)
- ผู้สนับสนุน (Supporter)

## Functional Requirements

### Admin Functions
- จัดการ User Management
- กำหนดระดับการดำเนินการ (Operation Level)
- บริหารจัดการ Observation Schedule 
- ดูแลเรื่อง shutdown time
- กำหนดเวลาทำการบำรุงรักษา (Maintenance)
- กำหนดเวลาที่ระบบเปิด-ปิด (System Open-Close time)
- ดูสถานะของอุปกรณ์และระบบ
- ดูข้อมูลเกี่ยวกับการตรวจสอบระบบ (system diagnostics)
- ควบคุม input และ output ของตัวระบบ

### Astronomer Functions
- ยื่นข้อเสนอโครงการวิจัยผ่านระบบ
- สร้าง Sci Plan
- ทดสอบ Sci Plan ก่อน submit
- Submit Sci Plan หลังจากทดสอบ
- เก็บข้อมูลผลลัพธ์การสังเกต
- ดู Live View ของกล้องโทรทรรศน์ 
- เข้าถึงข้อมูลการสังเกตที่ผ่านมา

### Telescope Operator
- สามารถเข้าใช้งานโทรทรรศน์ผ่าน Command line mode
- ระบบมีอินเตอร์เฟซระหว่าง Telescope Operator และ OCS Command line
- สามารถตรวจสอบโปรแกรมการสังเกตได้
- สามารถติดตั้ง Physical Unit เพิ่มได้
- สามารถตั้งค่าตารางการติดตั้งอุปกรณ์ได้

### Science Observer
- สามารถตรวจสอบ Science Plan ได้
- สามารถสร้าง Observing Program
- สามารถทดสอบ Observing Program 
- Execute Science Plan
- ตรวจสอบความคืบหน้าของโครงการที่กำลังทำงาน
- เก็บข้อมูลดาราศาสตร์ที่ได้มา
- ตรวจสอบความถูกต้องของข้อมูลทางดาราศาสตร์
- บริหารจัดการข้อมูลทางดาราศาสตร์ (เพิ่ม/ลบ/อนุญาต)
- ควบคุมโทรทรรศน์ผ่านโปรแกรมการสังเกต (OCS)

### Support
- สามารถติดตั้ง Extension เพิ่มเติมให้กับระบบ OCS เช่น ติดตั้ง algorithm ที่ Monitor ทดสอบระบบ OCS เพิ่มเติม
- ติดตั้ง Component ใหม่ให้กับระบบ OCS

## Non-functional Requirements

- มี Virtual Test Code สำหรับทดสอบการตั้งค่าและการทำงานของ Sci Plan ก่อนที่จะ submit
- ระบบ Gemini ต้องมีอายุการใช้งานที่ยาวนาน
- อินเตอร์เฟซผู้ใช้ควรสะท้อนถึงโหมดการเข้าถึงและระดับการทำงานอย่างชัดเจน
- เวลาในการเข้าถึงฐานข้อมูลจะอยู่ในช่วง 2-3 มิลลิวินาทีต่อการเข้าถึง
- ฐานข้อมูลจะต้องรองรับการเข้าถึงจากระยะไกลและในรูปแบบ distributed data
- รายละเอียดของโทรทรรศน์และอุปกรณ์ถูกเก็บไว้ในฐานข้อมูลออนไลน์เพื่อการใช้งานที่ง่าย
- ระบบควบคุมโทรทรรศน์ควรสามารถตรวจจับและเรียกใช้การทำงานแบบ parallel ได้
- การตรวจสอบควรทำการตรวจสอบแบบต่อเนื่องใน subsystem ที่ไม่ได้ทำงาน
- ต้องสามารถบันทึกข้อมูล log engineering ที่อัตราสูงถึง 200 Hz ในช่วงเวลาสั้นๆ
- เวลาในการ start-up ของ Telescope ใช้เวลาไม่เกิน 4 นาที
- มีระบบป้องกันการเข้าถึงเพื่อควบคุมฟีเจอร์ของระบบอาจจะจำกัดบางการทำงานให้เฉพาะสถานที่ระยะไกลบางแห่ง
- ระบบต้องรองรับการทำงานจากระยะไกลทั้งการสังเกตการณ์, ควบคุม telescope, ตู้ป้องกัน, และเครื่องมือ
- การวัดอัตราการเกิดข้อผิดพลาดในระหว่างการทดสอบเพื่อสร้าง baseline สำหรับการตรวจสอบความเชื่อถือของระบบ
